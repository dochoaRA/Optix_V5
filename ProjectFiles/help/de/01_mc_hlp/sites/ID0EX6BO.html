OPC-UA

ROVEMA OPC-UA Server
Der ROVEMA OPC-UA Server ist in die ROVEMA-Steuerung integriert.
Er stellt Maschinendaten (z. B. Messwerte, Sensordaten, Regelgrößen usw.) serviceorientiert und transparent bereit, auf die von einem kundenseitigen OPC-UA Client zugegriffen werden kann. Ein OPC-UA Client kann z. B. eine übergeordnete Visualisierung, ein SCADA-System oder auch eine fremde Steuerung sein.
Abb. 1: Topologie (Beispiel)
 
ROVEMA-Lieferumfang
Im ROVEMA-Lieferumfang enthalten ist der OPC-UA Server mit Lesezugriff auf alle standardmäßig für eine Maschine/Anlage bereitgestellten Daten.
Die Inbetriebnahme eines OPC-UA Clients sowie die dafür notwendigen Installationen und Einstellungen erfolgen kundenseitig.
 
Zusatzoptionen
Die folgenden Zusatzoptionen sind in Abstimmung mit ROVEMA erhältlich:
●
Generierung und Bereitstellung kundenspezifischer Zusatzdaten
 
●
Schreibrechte auf ausgewählte Daten
 
●
Rezepturvorgabe via Leitsystem:
Eine auf der Maschine abgelegte Rezeptur kann über das kundenseitige Leitsystem über eine Rezepturnummer angewählt werden. Einzelne Rezepturdaten werden dabei nicht übertragen.
 
●
OEE-Auswertung und Datenübertragung via OPC-UA:
Die Ergebnisse der OEE-Auswertung können im ROVEMA HMI auf einer separaten Seite eingesehen werden. Diese Daten werden zusätzlich über die OPC-UA Schnittstelle zur Verfügung gestellt.
 
 
Die Zusatzoptionen sind abhängig von dem Aufwand für die entsprechende Leistung zu vergüten.
 
Zugriffsarten
Die folgenden Zugriffsarten sind über einen OPC-UA Client möglich:
●
Daten lesen
 
●
Daten schreiben
 
 
Standardmäßig wird nur das Lesen der Daten unterstützt. Das Schreiben von Daten kann nur in Abstimmung mit ROVEMA erfolgen.
 
Einrichten/Verbindungsaufbau zum ROVEMA OPC-UA Server
Für das kundenseitige Einrichten bzw. den Verbindungsaufbau zum ROVEMA OPC-UA Server sind folgende Arbeitsschritte notwendig:
1.
Integration der ROVEMA-Maschine in das Kundennetzwerk:
 
a.
Verbindung zum Switch im Schaltschrank der ROVEMA-Maschine
 
b.
IP-Adresse im HMI einstellen (Vorgabe durch den Kunden)
 
 
2.
Test der Verbindung:
 
a.
Ein Verbindungstest mit einem OPC-UA Test-Client wird empfohlen. Der Download eines OPC-UA Test-Clients ist unter folgendem Link möglich:
https://www.unified-automation.com/downloads/opc-ua-clients.html.
 
b.
Es werden keine weiteren Programme für den Verbindungstest benötigt.

Sicherheit und Verschlüsselung
Für die Sicherheit und die damit zusammenhängende Verschlüsselung der Daten sind die folgenden Informationen und Vorgehensweisen zu beachten:
1.
Die Kommunikation erfolgt verschlüsselt mit 256sha256. Die Authentifizierung erfolgt über ein Benutzerprofil mit entsprechendem Passwort.
 
2.
Neues Zertifikat erstellen:
 
a.
Den Navigationspfad Inbetriebnahme ► System öffnen.
 
b.
In der Funktion OPC-UA den Schalter „Neues Zertifikat erstellen“ einschalten.
Nach Aktivieren des Schalters muss ein Neustart der Maschine erfolgen, um ein neues Zertifikat zu erstellen.
HINWEIS!
Ein neues Zertifikat muss erstellt werden, bevor das alte Zertifikat seine Gültigkeit verliert (Laufzeit 5 Jahre) oder wenn die IP-Adresse der Maschine geändert wurde.
 
 
 
3.
Zertifikat auf USB-Stick speichern:
 
●
Den Navigationspfad Inbetriebnahme ► System öffnen.
 
●
In der Funktion OPC-UA die Taste „Auf USB-Stick kopieren“ betätigen.
Das aktuelle Zertifikat wird auf dem angeschlossenen USB-Stick gespeichert.
HINWEIS!
Das Speichern des Zertifikats auf einem USB-Stick ist grundsätzlich nicht notwendig, da das Zertifikat i. d. R. automatisch an den OPC-UA Client übermittelt wird.
 
 
Bei technischen Rückfragen zur Sicherheit und Verschlüsselung kann der kundenseitige IT-Dienstleister bzw. die kundenseitige IT-Abteilung konsultiert werden.
 
4.
Verbindungsaufbau:
 
a.
Im gewählten OPC-UA Client die entsprechende IP-Adresse oder den Host-Namen eintragen, z. B. URL: opc.tcp://XXX.XXX.XXX.XXX (IP-Adresse)
 
b.
Während des Verbindungsaufbaus mit dem festgelegten Benutzerprofil und entsprechendem Passwort anmelden.
Abhängig von der OPC-UA Client Software können sich die dafür notwendigen Eingaben unterscheiden.
 
 
 
 
Standard-Daten
Die Daten in den unten stehenden Tabellen werden standardmäßig durch ROVEMA bereitgestellt. Bei Bedarf können in Abstimmung mit ROVEMA weitere kundenspezifische Zusatzdaten zur Verfügung gestellt werden.
Die Node IDs sind in OPC-UA in Namespaces eingeteilt. Für die allgemeinen Maschinendaten ist gemäß der OPC-UA Spezifikation der Namespace 1 reserviert.
 
NodeID-Liste erstellen
Eine Liste der OPC-UA-Daten kann als csv-Datei erstellt und auf einen USB-Stick kopiert werden.
Vorbereiten des USB-Sticks
1.
Alle Partitionen auf dem USB-Stick löschen. Der USB-Stick darf nicht partitioniert sein.
 
2.
Den USB-Stick mit dem FAT32-Dateisystem formatieren.
 
 
Liste mit NodeID‘s erstellen
1.
Über Navigationseintrag Inbetriebnahme und System in die Funktion OPC-UA gehen.
 
2.
In den Tab Einstellungen wechseln.
 
3.
Die Taste „Datenübersicht erstellen“ betätigen, um die NodeID-Liste zu erhalten.
 
 
Liste auf USB-Stick kopieren
1.
Den USB-Stick in die USB-Schnittstelle des PC‘s im Schaltschrank stecken.
 
2.
In den Tab Auf USB-Stick kopieren wechseln.
 
3.
Sobald die Zustandsanzeige „USB-Stick erkannt“ leuchtet, kann die Liste durch das Betätigen der Taste „Datenübersicht“ auf den USB-Stick kopiert werden.
 
4.
Nachdem das Wartesymbol erloschen ist, den USB-Stick aus der USB-Schnittstelle des PC‘s entfernen.
